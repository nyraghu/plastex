7 Families as functions

We have defined a family of types \begin{equation*}(Y_{x})_{x \mathrel{:}X}\end{equation*} to be an assignment that attaches to every element \(x \mathrel{:}X\) a type \(Y_{x}\), that is to say an element \(Y_{x}\) of the fixed universe \(\mathscr{U}\). According to the definition of a function in Section 4 above, this is just a function from \(X\) to \(\mathscr{U}\). The value \(Y_{x}\) of this function at any \(x \mathrel{:}X\) is the same as the value of the abstraction \begin{equation*}\lambda{}(x \mathrel{:}X).\, Y_{x}\end{equation*} at \(x\). So by \(\eta\)-conversion we have \begin{equation*}(Y_{x})_{x \mathrel{:}X} \equiv\lambda{}(x \mathrel{:}X).\, Y_{x}.\end{equation*}

From now we will view a family of types on a type \(X\) as a function \begin{equation*}F \mathrel{:}X \to\mathscr{U}.\end{equation*} This family would be written as \begin{equation*}(F(x))_{x \mathrel{:}X}\end{equation*} in the earlier notation.

One point that has been fudged above is that \(\mathscr{U}\) is not a type according to the definition that a type means an element of \(\mathscr{U}\). The statement \begin{equation*}\mathscr{U}\mathrel{:}\mathscr{U}\end{equation*} leads to a contradiction called Girardâ€™s paradox. So we have not yet defined the notion of a function from \(X\) to \(\mathscr{U}\).

To fill this gap suppose that \(\mathscr{U}\equiv\mathscr{U}_{i}\), and denote \(\mathscr{U}_{i+1}\) by \(\mathscr{U}_{+}\). The hierarchy property of universes implies that \(\mathscr{U}\mathrel{:}\mathscr{U}_{+}\). Everything we have said until now will be true if we take \(\mathscr{U}_{+}\) to be our fixed universe instead of \(\mathscr{U}\). But then \(\mathscr{U}\) would be a type because it is an element of the fixed universe \(\mathscr{U}_{+}\). The cumulativity of universes guarantees that the element \(X\) of \(\mathscr{U}\) is an element of \(\mathscr{U}_{+}\), so it is also a type with respect to \(\mathscr{U}_{+}\). The notion of a function from \(X\) to \(\mathscr{U}\) is now given by the definition, in Section 4, of a function from one type to another.
