15 Paths

Let us fix a type \(X\), and an element \(x \mathrel{:}X\). We are then given a family of types \begin{equation*}\operatorname{Path}_{x} \mathrel{:}X \to\mathscr{U}\end{equation*} on \(X\), called the path family of \(x\). For any element \(y \mathrel{:}X\), we will denote the type \begin{equation*}\operatorname{Path}_{x} (y) \mathrel{:}\mathscr{U}\end{equation*} by \begin{equation*}x = y.\end{equation*} Each of its elements is called a path from \(x\) to \(y\).

An element of the type \begin{equation*}x = x\end{equation*} is called a loop at \(x\). We are given a loop \begin{equation*}\epsilon_{x} \mathrel{:}x = x.\end{equation*} It is called the identity loop at \(x\).

For every element \begin{equation*}F \mathrel{:}\Pi{}(y \mathrel{:}X).\, x = y \to\mathscr{U},\end{equation*} and for every element \begin{equation*}a \mathrel{:}F(x)(\epsilon_{x}),\end{equation*} we are given an element \begin{equation*}\phi_{F,a} \mathrel{:}\Pi{}(y \mathrel{:}X) (p \mathrel{:}x = y).\, F(y)(p)\end{equation*} such that \begin{equation*}\phi_{F,a}(x)(\epsilon_{x}) \equiva.\end{equation*} We say that \(\phi_{F,a}(y)(p)\) is obtained by transporting \(a\) along the path \(p\) from \(x\) to \(y\).

We are using here the convention that the path operator has higher precedence than the arrow. So \begin{equation*}x = y \to\mathscr{U}\end{equation*} means \begin{equation*}(x = y) \to\mathscr{U},\end{equation*} not \begin{equation*}x = (y \to\mathscr{U}).\end{equation*}

We are also ignoring the problem that the object \begin{equation*}\Pi{}(y \mathrel{:}X).\, x = y \to\mathscr{U}\end{equation*} is not defined as \(\mathscr{U}\) is not an element of \(\mathscr{U}\). However, if \(\mathscr{U}\equiv\mathscr{U}_{i}\), and if we redo the earlier development with \(\mathscr{U}_{+}\) in the place of \(\mathscr{U}\), where \(\mathscr{U}_{+} \equiv\mathscr{U}_{i+1}\), then \begin{equation*}\Pi{}(y \mathrel{:}X).\, x = y \to\mathscr{U}\end{equation*} becomes a well-defined type for every \(y \mathrel{:}X\). This argument is like the one in Section 7 that justified the practice of looking at families as functions. We will skip such justification from now.

We can specialise the above discussion to the situation when \begin{equation*}F(y)(p) \equivG(y)\end{equation*} is independent of \(p\). So for every family of types \begin{equation*}G \mathrel{:}X \to\mathscr{U},\end{equation*} and every element \begin{equation*}a \mathrel{:}G(x),\end{equation*} we get an element \begin{equation*}\phi_{G,a} \mathrel{:}\Pi{}(y \mathrel{:}X).\, x = y \toG(y)\end{equation*} such that \begin{equation*}\phi_{G,a}(x)(\epsilon_{x}) \equiva.\end{equation*} Again \(\phi_{G,a}(y)(p)\) is called the element obtained by transporting \(a\) along \(p\).

Let us lastly look at the even more special case when \(G\) is a constant family. For every type \begin{equation*}H \mathrel{:}\mathscr{U},\end{equation*} and for every element \begin{equation*}a \mathrel{:}H,\end{equation*} we get an element \begin{equation*}\phi_{H,a} \mathrel{:}\Pi{}(y \mathrel{:}X).\, x = y \toH\end{equation*} such that \begin{equation*}\phi_{H,e}(x)(\epsilon_{x}) \equiva.\end{equation*} If \(p\) is a path from \(x\) to \(y\), we say then too that \(\phi_{H,a}(y)(p)\) is obtained by transporting \(a\) along \(p\).
